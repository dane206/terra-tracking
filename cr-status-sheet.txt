☑️ Phase 3 — How each event is verified (repeatable checklist)
For each event, we will do all three checks:

B. Verify BigQuery (authoritative)
Example (replace EVENT_NAME):
```
bq query --project_id=terra-analytics-dev --use_legacy_sql=false \
'SELECT
  received_at,
  data_source,
  event_name,
  event_id,
  event_time,

 LIMIT 1'
```

An event is DONE when this is true - The same user action produces the same event name, the same meaning, and the same row in BigQuery every time, and changing it would require a conscious migration.

For any event, you do these checks once:
- Name is final - The event name will never change again.
- Meaning is final - Everyone agrees what user action it represents.
- It fires once per action - No retries, no accidental doubles.
- It survives the pipeline unchanged dataLayer → GTM → ingest → BigQuery → GA4 (same event, same semantics)
- You can point to one place where it’s emitted - One source of truth for firing logic.
When all five are true → you never reopen that event again.

raw contains full context (page, item, checkout, etc.)

C. Verify GA4 DebugView
Event appears
Parameters match expectation

No missing required fields
